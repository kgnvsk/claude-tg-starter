# Субагенты этого бота

Здесь живут **субагенты** — специализированные под-исполнители, которых бот вызывает
под отдельные задачи (ресёрч, ревью кода, что угодно). Это нативная фича Claude Code:
**один `.md`-файл = один агент.** Их токены идут на ту же подписку, что и у бота —
отдельный метерящийся пул не задействуется.

## Как добавить своего агента

1. Создай файл `имя-агента.md` в этой папке (`~/.claude/agents/`).
2. Впиши формат (ниже).
3. **Перезапусти бота**, чтобы он увидел нового агента:
   `sudo systemctl restart claude-telegram`
   Пока не рестартнёшь — новый файл НЕ подхватится.

## Формат файла

    ---
    name: имя-агента          # обязательно: маленькие буквы и дефисы
    description: Когда меня звать — 1-2 предложения. Бот читает это и сам решает делегировать.   # обязательно
    tools: WebSearch, WebFetch, Read, Grep, Glob   # опционально: через запятую (НЕ список). Пусто = все инструменты.
    model: sonnet             # опционально: sonnet | opus | haiku | inherit (по умолчанию inherit)
    ---

    Здесь — системный промпт агента: кто он, как работает, что возвращает.

Обязательны только `name` и `description`. Остальное опционально.

## Как бот их вызывает

- **Сам**, когда твоя задача совпала с `description` агента.
- **По имени**, если попросишь напрямую: «возьми researcher и собери инфу про …».

## Важно про лимит

Каждый субагент — полноценная под-сессия. Работает на подписке (не на отдельном
платном пуле), но **ест лимитное окно быстрее** обычного чата. Для лёгких задач
ставь агенту `model: haiku` — дешевле по окну.

## Пример

Рядом лежит `researcher.md` — рабочий веб-ресёрч-агент. Скопируй его как шаблон под своего.
